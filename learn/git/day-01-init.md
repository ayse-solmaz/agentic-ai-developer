# Day 1 — Initializing a Repository

## What I did
1. Created a project directory (`git-day1-practice`)
2. Navigated into it
3. Ran `git init` to start tracking
4. Verified `.git` folder with `ls -Force`

## Commands
```powershell
mkdir git-day1-practice
cd git-day1-practice
git init
ls -Force
```

## Key idea
`git init` creates a hidden `.git` folder — Git stores all history and metadata there.

## Lesson learned
A folder inside an existing repo is NOT its own repo unless it has its own `.git`.
