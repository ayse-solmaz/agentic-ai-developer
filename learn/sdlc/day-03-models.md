# Day 3 — Waterfall, Agile, and Iterative Models

## Waterfall sketch

**Phases (linear order):** Requirements → Design → Implementation → Testing → Deployment → Maintenance

**Strengths:**
1. Good for fixed, regulated projects (e.g. banking, legal).
2. Easy to plan and document with clear milestones.

**Weaknesses:**
1. Feedback comes too late.
2. Changing scope later is expensive.

## Agile sketch

**Short iterations:** Work happens in short cycles called sprints (1–4 weeks).

**Feedback:** Users give feedback after every sprint.

**Changing priorities:** Backlog is reordered before each sprint.

**Key terms:**
- **Iteration** — one short repeating work cycle.
- **Sprint** — a fixed-length iteration with a clear goal.
- **Backlog** — prioritized list of work items.

## Compare for a project

| Project | Which model? | Why? |
|---------|--------------|------|
| Bank core system | Waterfall | Rules are fixed, mistakes are costly. |
| Marketing landing page | Agile | Needs change fast, quick feedback matters. |

## Hybrid reality

Most teams mix both models. Core architecture is planned upfront like Waterfall. Features are built in Agile sprints. Before release, a final test gate checks everything works.
